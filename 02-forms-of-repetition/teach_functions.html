What Is a Function?

You've already been using them. Now you'll write your own.

Step 1

You've been calling functions this whole time

Every time you write background() or line() or rotate(), you are calling a function. A function is a named action — a package of instructions you can trigger by name.

createCanvas() background() stroke() strokeWeight() line() translate() rotate() push() pop() noLoop()

p5.js wrote all of those for you. When you type line(0, 0, 200, 200), you're not writing the line-drawing code — you're telling p5 to run it.

A function call = a name + parentheses.
The values you put inside the parentheses are called arguments. They are the inputs the function needs to do its job.
Step 2

You've also been writing functions

setup() and draw() are functions you wrote. p5.js calls them for you automatically — once and then on a loop. Let's look at the anatomy of a function definition:

anatomy of a function
function setup() {
  // everything inside the { } runs when this function is called
  createCanvas(400, 400);
  noLoop();
}
function The keyword that says "I'm defining a function"
setup The name — this is how you call it later
() The parentheses — inputs go here (this one takes none)
{ … } The body — all the instructions that run when the function is called
Step 3

Writing your own function

You can define a function with any name and then call it wherever you want. Here's a function that draws one diagonal line across a 200×200 tile:

defining the function
function drawTile() {
  line(0, 0, 200, 200);
}
Defining a function doesn't run it. It just gives a name to a set of instructions. The instructions only run when you call the function.
calling the function
function draw() {
  background(20, 20, 40);

  // calling drawTile() runs whatever is inside it
  drawTile();
}

function drawTile() {
  line(0, 0, 200, 200);
}
Step 4

Where your function fits into the grid

The grid engine uses the double for loop you already know, plus push(), translate(), and rotate() to place each tile. drawTile() is called once per cell — it's the only place the tile is described.

the engine — drawTile() is called inside the loop
for (let r = 0; r < rows; r++) {
  for (let c = 0; c < cols; c++) {

    push();

    // move to this tile's position on the canvas
    translate(c * cellSize, r * cellSize);

    // rotate the tile randomly 0°, 90°, 180°, or 270°
    rotate(floor(random(4)) * HALF_PI);

    drawTile();  ← your function runs here, once per cell

    pop();
  }
}

// ─────────────────────────────────────────
// This is the only place the tile is described.
// Change what's inside and the whole grid changes.
function drawTile() {
  line(0, 0, cellSize, cellSize);
}
The engine is the engine. The tile is yours.
Change drawTile(). Change everything.
