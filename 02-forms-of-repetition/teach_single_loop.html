The For Loop

When you find yourself writing the same line again and again — that's a loop waiting to happen.

Step 1

Here's what we want to draw

Four squares in a row across the canvas, spaced evenly. This is our goal.

goal — 4 squares
Step 2

How do we do it? Write it out

Start with a variable x at the first position. Draw a square. Move x forward. Repeat. Let's write it out once for each square so we can see the pattern:

single_loop_manual.js
let x = 20;

rect(x, 60, 80, 80);  // square 1, x = 20
x += 100;

rect(x, 60, 80, 80);  // square 2, x = 120
x += 100;

rect(x, 60, 80, 80);  // square 3, x = 220
x += 100;

rect(x, 60, 80, 80);  // square 4, x = 320

Notice the two lines that repeat every time:

the pattern
rect(x, 60, 80, 80);
x += 100;

rect(x, 60, 80, 80);
x += 100;

rect(x, 60, 80, 80);
x += 100;
…
Any time you see the same block copy-pasted, a loop can replace it.
Step 3

Collapse it into a for loop

We want to run those two lines 4 times. A for loop does exactly that — it repeats a block a set number of times and keeps count for you.

single_loop_for.js
let x = 20;

for (let i = 0; i < 4; i++) {
  rect(x, 60, 80, 80);
  x += 100;
}

The two lines inside the loop are exactly what we wrote four times by hand. Same code. Same result. Less typing.

let i = 0
Start: set the counter to 0
i < 4
Condition: keep going while i is less than 4
i++
Update: add 1 to i after each pass
i counts the passes: 0, 1, 2, 3. When i reaches 4, the condition i < 4 is false and the loop stops. That's exactly 4 runs.

What's next

One loop → one row. Two loops → a full grid.

This loop moves across one row. If we wrap it inside another loop that moves down the rows, we get every cell in a grid — and that's exactly how the Truchet engine works.

preview — the double for loop
// Outer loop: moves down the rows
for (let r = 0; r < num_rows; r++) {

  // Inner loop: moves across the columns
  for (let c = 0; c < num_cols; c++) {
    rect(c * col_width, r * row_height, col_width, row_height);
  }
}
