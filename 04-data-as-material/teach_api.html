🔌 What Is an API?

API = Application Programming Interface

A way for your code to talk to another computer and ask for data.

Think of it like ordering from a restaurant menu. You don't go into the kitchen — you just place your order, and the kitchen sends back your food.

🔗 Breaking Down the URL

Each & adds another item to your order:

https://api.open-meteo.com/v1/forecast ← the "restaurant" ?latitude=40.65&longitude=-73.95 ← where (Brooklyn) &hourly=precipitation,rain ← what hourly data &daily=sunrise,sunset ← what daily data &timezone=auto ← when &forecast_days=7 ← how much &temperature_unit=fahrenheit ← preferences

📦 What Comes Back?

Week 4 (API data)

168 data points (24h × 7 days)

✅ Same shape — just bigger. Still objects, still arrays, still dot notation.

🔍 Accessing the Data

Chain dots to dig into nested objects:

// The full response object data // The hourly section data.hourly // The precipitation array (168 numbers) data.hourly.precipitation // The first hour's rainfall data.hourly.precipitation[0] // → 0.0

Same dot notation as sleepData.hours — just deeper!

⏳ Loading Data in p5.js

New function: preload()

let data; function preload() { data = loadJSON(apiURL); // fetch the data from the API } function setup() { // data is ready here — guaranteed! } function draw() { // use data freely }
Why preload? It pauses everything until the data arrives. Without it, setup() might run before the data is ready → crash.
